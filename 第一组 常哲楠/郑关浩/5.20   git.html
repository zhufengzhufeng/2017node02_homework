5.20 git

1.配置git用户和邮箱
2.git config --global user.name "你的github用户名"
3.git config --global user.email "你的github邮箱"
4.输入git init命令把当前这个目录变成git可以管理的仓库
5.通过 ls -al 查看所有文件
6.git add .   添加到暂存区
7.git commit -m "注释内容"  添加到历史区
8.git log  查看历史状态
9.git diff 比较工作区和暂存区的不同
10.git checkout "文件名"  从暂存区覆盖工作区内容
11.git reset --hard id  从历史区覆盖工作区（id最少7位数）
12.git reflog  查看所有历史版本
13.git reset HEAD "文件名"   删除上一次添加到暂存区的内容
1.分支
2.git branch
3.git branch 创建分支
4.git checkout a
5.git checkout -b c切换分支
6.在master  git merge
7.git checkout b
8.git branch --merged 合并了哪些分支
9.git branch --no-merged 合并了哪些分支
10.git branch -d a 删除分支
11.git branch -D a 删除分支
12.git checkout -b 分支名   创建并切换
13.默认我们的代码是放在工作区上，不属于任何分支，只有提交到某个分支上，此文件才归属于某个分支
1.合并分支
2.git merge "分支名"  合并
3.合并多个分支时可能合并的内容会产生冲突，手动解决冲突，再提交最新结果
4.git pull origin master  获取最新代码

1.fork
2.- 叉子 将别人的代码当前的状态，克隆一份放到自己的仓库上，一个项目只能fork一次，代码更新之间不会影响
3.- 克隆  是将线上的项目拉取到本地，拉下来后就是git仓库，而且已经添加了远程仓库地址
4.1.打开组长git地址，点击fork；
5.2.当看到名字变成自己的名字后，说明fork成功![Alt text](./1495270335578.png)
6.3.复制地址
7.4.在本地打开git bash ，`git clone 地址`可以在地址前起别名，也可以不起，不起的话，默认文件夹名称是组长的仓库的名称
8.5.`git add .`添加文件

  • 闭包
  • 单例：不能完全解决命名冲突

node特点

  • 异步io (定时器。回调函数)
  • 单线程(其他语言多线程) 多线程实现就是切换执行上下文
  • 进程是包含线程的，node一个进程只能开一个线程

如何解决异步问题

  • 当我们调用一个异步方法时，传入后续的逻辑的函数，作为参数 (回调函数)
